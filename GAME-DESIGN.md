# 🏥 「规培生存指南」—— 微信小游戏设计规格书

> **版本**: v1.0 MVP | **日期**: 2026-07-10 | **作者**: 暴走の规培医生

---

## 一、游戏概述

### 1.1 一句话描述

一款面向医学生/规培生的微信小游戏：通过分支叙事选择推进剧情，融合真实医学知识和医疗圈幽默梗，不同选择导向不同结局。

### 1.2 核心体验

玩家扮演刚入职的规培医生，在急诊科度过第一个轮转月。每个选择都在专业与人情之间拉扯，同时在"水果禁忌""闲者必忙"等医疗圈潜规则的包围中艰难求生。

### 1.3 对标参考

《盛世天下》—— 选择导向、分支叙事、多结局、属性累积

### 1.4 目标用户

- 医学生（本科/研究生阶段）
- 规培生/住院医师
- 医学院毕业生（能get医疗圈内部梗）

## 二、游戏机制

### 2.1 核心循环

```
新游戏 → 属性初始化 → 场景1 → [选择A/B/C] → 场景N → ... → 结局判定 → 结局画面
                                                     ↓
                                               梗向必死结局
                                               (当场触发, 可回溯)
```

### 2.2 属性系统

四项隐藏属性，0-100 区间，起始值如下：

| 属性 | 起始值 | 含义 | 受什么选择影响 |
|------|--------|------|---------------|
| 专业度 | 50 | 临床知识、诊断准确性 | 医学相关的诊断/处置选择 |
| 人缘 | 50 | 同事口碑、患者满意度 | 社交选择、帮忙与否 |
| 体力 | 80 | 精力与健康 | 随时间递减；休息选择可恢复 |
| 运气 | 40 | 不可控因素 | 水果/穿搭/禁忌选择；随机事件 |

**设计原则**：
- 属性变化对玩家不可见（不像RPG那样显示数值）
- 但通过角色台词给暗示（"你今天脸色不太好" ← 体力低）
- 结局判定时才揭晓最终属性

### 2.3 选择系统

每个场景提供 2-3 个选项，分为三类：

| 类型 | 示例 | 效果 |
|------|------|------|
| **专业选择** | 胸痛病人 → "立刻做心电图" vs "先问病史" | 影响专业度 |
| **社交选择** | 同事求助 → "帮忙" vs "太忙了" | 影响人缘 |
| **禁忌选择** | 吃芒果/火龙果/穿红色… | 触发梗结局 或 影响运气 |

### 2.4 结局系统

结局分为两类：

#### A. 梗向必死结局（即时触发，喜剧向）

玩家踩中医疗圈禁忌 → 立即触发短结局 → 回到选择点

| # | 结局名 | 触发条件 | 结局画面 |
|---|--------|---------|---------|
| 🥭 | 芒果之夜 | 接受/吃了芒果 | "今晚急诊量暴涨300%，天亮了你的腿还在发抖" |
| 🐉 | 火龙果之约 | 吃了火龙果 | "连收3个STEMI+2个脑卒中，导管室今晚属于你" |
| 🧧 | 旺仔魔咒 | 喝了旺仔牛奶 | "病人多到加床排到走廊尽头，护长意味深长地看着你" |
| 🍓 | 草莓味夜班 | 吃了草莓 | "医嘱被退回3次，带教的眼神能杀人" |
| 🫐 | Blue Code | 吃了蓝莓 | "Code Blue响了整整一夜，心肺复苏做到手臂抽筋" |
| 🍇 | 杨梅运 | 吃了杨梅 | "针打不进去、化验单找不到、药房说没收到处方" |
| 👗 | 大红大紫 | 穿了红色或紫色 | "连收了3个消化道大出血+2个过敏性紫癜" |
| 😴 | 闲者的惩罚 | 感叹"今晚好闲"或做闲事 | "话音刚落，3辆救护车同时停在了急诊门口" |
| 🙊 | 死神之问 | 问了"忙不忙" | "问了就不用来上班了——因为今晚会忙到你想辞职" |

> 每个梗结局后显示：「这是第X/7个梗结局」+ 提示"再试试别的选择？"

#### B. 属性累积结局（通关时判定）

| # | 结局名 | 判定条件 | 结局文本概述 |
|---|--------|---------|-------------|
| ⭐ | 急诊新星 | 专业度≥75, 人缘≥50 | "主任拍拍你的肩膀：下个月继续加油，我看好你。" |
| 🤝 | 最佳搭档 | 人缘≥80, 专业度≥50 | "护士长说：你是第一个能把急诊科哄得团团转的规培生。" |
| 😅 | 勉强过关 | 专业度≥30, 人缘≥30, 体力≥20 | "这个月总算过完了，虽然灰头土脸，但你活下来了。" |
| 💀 | 医务科约谈 | 专业度<25 或 人缘<20 | "病人投诉信已经放在医务科桌上，你被叫去谈话。" |
| 🏥 | 过劳晕倒 | 体力<10 | "你在查房时眼前一黑，醒来时发现自己躺在值班室的床上，手上挂着葡萄糖。" |

## 四、技术方案

### 4.1 技术栈

| 层 | 选型 | 理由 |
|----|------|------|
| 平台 | 微信小游戏 | 目标平台 |
| 语言 | ES6 JavaScript | 原生支持，零依赖 |
| 渲染 | Canvas 2D API | 图文叙事不需要复杂渲染 |
| 数据 | JSON 配置文件 | 加场景只改配置，不改代码 |
| 存储 | wx.setStorageSync | 存档 + 结局收集 |
| 构建 | 无 | 微信开发者工具直接运行 |

### 4.2 项目结构

```
医学游戏/
├── game.js                 # 微信小游戏入口
├── game.json               # 微信小游戏配置
├── project.config.json     # 开发者工具项目配置
├── src/
│   ├── engine.js           # 核心状态机：场景→选择→属性→跳转
│   ├── renderer.js         # Canvas 渲染：文本、选项按钮、简单动画
│   ├── storage.js          # 存档与结局收集管理
│   └── data/
│       ├── episode-1.js    # 第一集「急诊科」场景数据
│       ├── characters.js   # 角色定义
│       └── endings.js      # 结局文本与条件
└── README.md
```

### 4.3 数据设计（核心）

场景数据用 JS 对象（比 JSON 灵活，支持注释和简单逻辑）：

```javascript
// episode-1.js 示例结构
const episode1 = {
  id: 'ep1',
  title: '急诊科',
  scenes: {
    start: {
      id: 'start',
      text: '今天是规培第一天。推开急诊科的门，消毒水味扑面而来。',
      choices: [
        { text: '主动跟主任去查房', next: 'scene_ward', attr: { 专业度: 5 } },
        { text: '跟着带教老师去急诊区', next: 'scene_er', attr: { 人缘: 5 } },
        { text: '先看排班表熟悉环境', next: 'scene_schedule', attr: { 体力: 5 } }
      ]
    },
    // ...更多场景
  }
};
```

### 4.4 属性系统实现

```javascript
// engine.js
const attributes = {
  专业度: 50,
  人缘: 50,
  体力: 80,
  运气: 40
};

function applyChoice(choice) {
  // 应用属性变化
  Object.entries(choice.attr).forEach(([key, delta]) => {
    attributes[key] = Math.max(0, Math.min(100, attributes[key] + delta));
  });
  // 检查是否触发梗结局
  if (choice.jokeEnding) {
    return showJokeEnding(choice.jokeEnding);
  }
  // 跳转下一场景
  return loadScene(choice.next);
}
```

## 五、MVP 范围

### 包含
- [x] 第一集「急诊科」10 个场景
- [x] 4 个属性累积结局
- [x] 9 个梗向必死结局（水果6 + 穿搭1 + 闲者2 + 死神之问1，复用场景4/6）
- [x] 3-6 个角色（文字描述，MVP 不配立绘）
- [x] 存档功能（保存当前进度）
- [x] 结局收集画廊（解锁过的结局可重看）

### 不包含（第二集及以后）
- 第二集「内科」及后续剧集
- 角色立绘和场景图
- 音效和背景音乐
- 排行榜、社交分享
- 体力/付费系统
- 微信审核发布
